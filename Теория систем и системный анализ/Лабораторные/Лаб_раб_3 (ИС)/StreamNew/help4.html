Как работать с моделью в AnyLogic

Если при построении модели не возникли ошибки, то, в зависимости от выбора пользователя, происходит следующее:

  • создается файл AnyLogic, состоящий из пустого объекта Main (корневого объекта модели) и объекта Stream1 (рис. 5), представляющего введенную схему, или
  • новый объект с именем StreamN (N – номер, обеспечивающий уникальность имени объекта) добавляется к уже существующей модели.

Рисунок 5.

При этом входными переменным объекта являются параметры источников тока и напряжения. Параметрами модели являются величины емкостей конденсаторов, сопротивлений резисторов, индуктивностей катушек (параметры имеют имена CN, RN, LN соответственно, где N – номер элемента схемы). Начальные значения элементов C и L также являются параметрами модели (имена вида VN).

Запуск моделирования

Вставьте в корневой объект модели (Main) один объект класса Stream1 (рис. 6). Это можно сделать простым перетаскиванием Stream1 из дерева структуры модели в область объекта Main. Вставленному объекту можно дать новое имя – он в любом случае останется экземпляром класса Stream1.

Рисунок 6.

Перед запуском моделирования необходимо создать новый эксперимент. Для этого щелкните правой кнопкой мыши в дереве структуры модели по значку Эксперименты и в выпадающем меню выберите Новый эксперимент. Теперь можно начинать моделирование кнопкой Запустить (F5).

Изменение параметров и входных данных модели

Перед запуском моделирования может возникнуть необходимость изменить параметры элементов модели или их начальные значения. Это можно сделать в окне свойств объекта-схемы (вернее того экземпляра класса Stream1, который был вставлен в Main – изменять же параметры у класса объектов нужно в свойствах класса объектов Stream1). Вид окна свойств приведен на рис. 7.

В созданной модели имеются входные переменные – функции силы тока и напряжения соответствующих элементов. По умолчанию им присвоены константные значения. Однако, согласно концепции AnyLogic, на них может быть подан любой сигнал с переменных или других объектов. Для этого необходимо соединить коннектором нужную выходную переменную со входной переменной объекта. Этот прием продемонстрирован на рис. 8.

Просмотр внутренней структуры модели и ее уравнений

Чтобы просмотреть внутреннюю структуру модели, дважды щелкните мышью на классе Stream1 в дереве структуры модели. При этом откроется окно содержимого выделенного класса объектов (рис. 9). Переменные вида ui означают разность потенциалов на i-м элементе схемы, а переменные вида ii – силу тока, проходящего через i-ый элемент

Рисунок 9.

Чтобы просмотреть систему уравнений, описывающих схему, щелкните на значке Код в объекте схемы. Окно кода представлено на рис. 10.
